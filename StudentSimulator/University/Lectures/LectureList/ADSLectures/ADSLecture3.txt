==========Лекція №3 - Експоненційний пошук==========

Експоненційний пошук використовується для пошуку в дуже 
великих відсортованих масивах.

Ідея:

👉 спочатку швидко знайти діапазон, де може бути число
👉 потім виконати бінарний пошук у цьому діапазоні.

Кроки:

Перевіряємо індекси: 1, 2, 4, 8, 16, 32…
Поки не “перестрибнемо” потрібне число.
Потім робимо бінарний пошук у знайденому діапазоні.

Це як швидко шукати сторінку в книзі:
спочатку грубо, потім точно.

Складність: O(log n)

Уяви, що шукаєш слово в дуже товстій книзі.

Ти не відкриваєш кожну сторінку:

Спочатку швидко перегортаєш сторінки великими стрибками
Коли приблизно знайшов розділ — шукаєш точно

Саме так працює експоненційний пошук.

Детальніше:

1. Спочатку перевіряємо:

if array[0] == target → return 0

Бо шуканий елемент може бути на початку.

2. Пошук діапазону (експоненційний ріст)

Починаємо з індексу 1 і подвоюємо його:

index = 1
index = 2
index = 4
index = 8
index = 16
index = 32

Поки:

index < n && array[index] <= target

Ми ростемо експоненційно (×2).

Тому алгоритм називається експоненційний.

Ми знайшли діапазон:

target ∈ (index/2 , index)

Тобто число точно лежить між двома степенями двійки.

Наприклад:

Індекси перевірки: 1,2,4,8,16,32
Перестрибнули число на 32

→ Шукаємо між 16 і 32

Тепер запускаємо бінарний пошук у цьому діапазоні.
