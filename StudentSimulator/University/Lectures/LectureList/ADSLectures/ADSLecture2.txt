==========Лекція №2 - Інтерполяційний пошук==========

Інтерполяційний пошук на відміну від бінарного намагається
вгадати позицію числа

Як працює:
Якщо масив рівномірний (наприклад 10,20,30,40…),
алгоритм одразу стрибає приблизно туди, де має бути число.

Уяви телефонний довідник:
ми відкриваємо не середину, а приблизну букву.

Складність:

Кращий випадок: O(log log n) 🤯
Гірший випадок: O(n)

Працює добре лише для рівномірно розподілених даних!

Детальніше:

Межі на початку ті самі що і в бінарного пошуку:
left = 0
right = array.Length - 1;

Але середину беремо інакше. Замість mid = (left + right) / 2:

pos = left + ((x - arr[left]) * (right - left)) / (arr[right] - arr[left])

x - шуканий елемент

Це формула лінійної інтерполяції.

Що значить кожна частина?

(x - arr[left]) - наскільки шукане число далеко від лівої межі
(arr[right] - arr[left]) - розмір діапазону значень
(right - left) - розмір діапазону індексів

Працюємо поки:

while (left <= right && target >= array[left] && target <= array[right])
